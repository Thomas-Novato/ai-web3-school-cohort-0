## 学习内容

### 为什么要学 Network

很多 Web3 问题表面上是前端或钱包问题，底层其实是网络问题：切错链、RPC 延迟、交易 pending、测试网资产不足、L2 提现等待。理解 Network，是为了知道一笔交易从签名到最终确认，中间经过了哪些层：

> **钱包 → RPC → Mempool → 区块 → 共识 → 执行 → 确认 → 索引 → 浏览器**

### 第一性原理

区块链网络的核心不是"存数据"，而是让**互不信任的参与者对状态变化达成一致**。

| 关键点 | 说明 |
|--------|------|
| **区块是同步节奏** | 状态不是连续更新，而是按区块批量推进 |
| **共识是信任来源** | 用户相信状态，因为网络按规则达成一致，不是某家公司说了算 |
| **网络选择影响体验** | 主网、测试网、L2 的费用、速度、安全假设都不同 |

### 知识节点

#### Block（区块）— 初级
区块是交易被批量提交和排序的单位。关注三件事：
- ✅ 交易有顺序，顺序影响结果
- ✅ 区块有 gas limit，网络吞吐不是无限的
- ✅ 新区块引用前一区块，形成可验证历史

#### Consensus（共识）— 中级
网络决定"哪段历史有效"的机制。对应用开发者来说：
- 交易需要等几个 confirmation 才算安全
- 区块可能短暂重组，前端不能过早假设最终结果
- 节点故障或 RPC 异常时，状态读取可能延迟

#### PoS（权益证明）— 中级
以太坊当前使用 PoS。验证者质押 ETH 参与区块提议和证明。安全来自**经济质押、客户端实现和节点参与**，不是"免费"的。

#### Testnet（测试网）— 初级
在接近真实链的环境里测试合约、前端和交易流程。做项目时至少记录：
- 部署在哪条测试网
- 合约地址和部署交易
- 使用哪个 RPC
- 前端如何切换 chain id
- 测试资产从哪里获取

#### L2（二层网络）— 中级
在主网安全和更低成本之间做扩展。费用更低、确认更快，但多了**桥、提现等待、排序器和跨链状态同步**等复杂性。

#### Rollup — 高级
主流 L2 扩展路线，把执行搬到链下，再把结果提交到 L1。包括 optimistic rollup 和 ZK rollup。Rollup 降低了单笔交易成本，但没有消除链上系统复杂度。

### 在 AI × Web3 中的位置

AI Agent 如果要读取链上状态或执行交易，必须知道自己在哪条网络上操作。主网和测试网、L1 和 L2、不同 chain id、不同合约地址，**不能靠模型"猜"**。

更稳妥的做法是让工具返回结构化网络信息：chain id、RPC 来源、区块高度、交易哈希、确认数、explorer 链接。Agent 的总结应该引用这些可验证信息。

### 与之前 AI 知识的关联

| 关联概念 | 说明 |
|----------|------|
| **Context 章节** | Network 信息（chain id、RPC 来源）应作为结构化上下文进入 Agent |
| **Agent 章节** | Agent 执行链上操作时，网络选择影响工具调用的结果和安全边界 |
| **MCP 章节** | MCP Server 暴露网络相关工具时，需要明确 chain id 和网络类型 |
| **AI Security** | 网络选择错误（如测试网当主网）是安全隐患之一 |

- 区块链网络不是抽象背景，而是交易从签名到确认的完整链路
