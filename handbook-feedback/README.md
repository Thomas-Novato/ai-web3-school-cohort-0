# Handbook Feedback

这个目录用于记录在学习 AI × Web3 School Handbook 过程中发现的问题、建议和改进意见。

## 为什么要记录 Feedback

- 帮助改进 Handbook 质量
- 让后续学员受益
- 沉淀学习过程中的思考
- 建立可索引、可复盘的学习材料

## Feedback 格式

每条 feedback 建议包含：

1. **Handbook 页面链接**: 具体到某个章节
2. **问题类型**: 
   - 内容错误（错别字、技术错误）
   - 内容过期（链接失效、版本过时）
   - 概念不清（解释不够清楚）
   - 结构建议（章节组织、导航）
   - 缺失内容（需要补充的知识点）
3. **问题描述**: 详细说明发现的问题
4. **建议改进**: 如何改进会更好
5. **来源**: 学习日期、相关笔记链接

## 示例

### feedback-2026-05-20-llm-chapter.md

```markdown
# Feedback: LLM 章节

**页面链接**: https://aiweb3.school/zh/handbook/ai/llm/

**问题类型**: 概念不清

**问题描述**: 
在"模型能力边界"部分，提到了"模型不能替代什么"，但没有具体举例说明在 AI × Web3 场景下的边界案例。

**建议改进**:
可以增加一些 AI × Web3 场景的具体例子，比如：
- Agent 不应该在没有人工确认的情况下执行大额转账
- 模型不能替代智能合约的安全审计
- Agent 不能自主决定关键的治理投票

**来源**: 
- 学习日期: 2026-05-20
- 相关笔记: daily/2026-05-20.md
```

## 提交 Feedback

1. 在本地记录 feedback（handbook-feedback/ 目录）
2. 定期整理和提交到 GitHub
3. 可以通过 GitHub Issues 或 PR 提交到官方仓库

## 注意事项

- Feedback 应该是建设性的
- 尽量提供具体的改进建议
- 如果不确定是否是问题，也可以记录下来作为疑问
